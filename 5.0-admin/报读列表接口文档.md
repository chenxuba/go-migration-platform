# 报读列表接口文档

## 接口信息

- **接口路径**: `/api/saleOrder/getRegistrationListPage`
- **请求方式**: POST
- **接口描述**: 查询报读列表数据（学费账户列表）

## 重要字段说明

### 数量字段

1. **totalQuantity**: 总购买数量（不含赠送）
   - 购买部分的总数量
   - 不包含赠送的课时/天数

2. **totalFreeQuantity**: 总赠送数量
   - 赠送部分的总数量

3. **quantity**: 剩余购买数量（不含赠送）
   - 计算公式：`MAX(0, remainingQuantity - totalFreeQuantity)`
   - 表示还剩多少购买的课时/天数

4. **freeQuantity**: 剩余赠送数量
   - 计算公式：`MIN(totalFreeQuantity, remainingQuantity)`
   - 表示还剩多少赠送的课时/天数

### 有效期字段

1. **validDate**: 有效期开始日期
   - 购买部分的开始日期
   - 格式：yyyy-MM-dd

2. **endDate**: 有效期结束日期（含赠送部分）
   - 如果有赠送，显示赠送部分的结束日期
   - 如果没有赠送，显示购买部分的结束日期
   - 格式：yyyy-MM-dd

## 数据存储说明

### 学费账户拆分规则

有赠送的订单会拆分成两条学费账户记录：

1. **购买部分记录**：
   - `validDate`: 购买开始日期（如 2026-01-18）
   - `endDate`: 购买结束日期（如 2026-03-17）
   - `totalQuantity`: 购买天数（如 59天）
   - `freeQuantity`: 0
   - `remainingQuantity`: 购买天数
   - 承担全部金额

2. **赠送部分记录**：
   - `validDate`: 赠送开始日期（购买结束日期+1天，如 2026-03-18）
   - `endDate`: 赠送结束日期（如 2026-03-21）
   - `totalQuantity`: 0
   - `freeQuantity`: 赠送天数（如 4天）
   - `remainingQuantity`: 赠送天数
   - 金额为 0

### 查询合并规则

查询报读列表时，会按 `order_course_detail_id` 分组，将购买和赠送两条记录合并成一条：

- `totalQuantity`: SUM(购买部分的 totalQuantity) = 59天
- `totalFreeQuantity`: SUM(赠送部分的 freeQuantity) = 4天
- `validDate`: MIN(validDate) = 2026-01-18（购买开始日期）
- `endDate`: MAX(endDate) = 2026-03-21（赠送结束日期）

### ES 同步规则

- 只同步购买部分的记录到 ES
- 赠送部分的记录（`totalQuantity=0` 且 `freeQuantity>0`）不同步到 ES
- ES 查询时直接返回购买部分的记录，无需合并

## 示例

### 示例1：有赠送的订单

**订单信息**：
- 购买：2个月（2026-01-18 到 2026-03-17，共59天）
- 赠送：4天（2026-03-18 到 2026-03-21）

**数据库存储**（2条记录）：

记录1（购买部分）：
```json
{
  "validDate": "2026-01-18",
  "endDate": "2026-03-17",
  "totalQuantity": 59,
  "freeQuantity": 0,
  "remainingQuantity": 59
}
```

记录2（赠送部分）：
```json
{
  "validDate": "2026-03-18",
  "endDate": "2026-03-21",
  "totalQuantity": 0,
  "freeQuantity": 4,
  "remainingQuantity": 4
}
```

**接口返回**（合并后1条记录）：
```json
{
  "validDate": "2026-01-18",
  "endDate": "2026-03-21",
  "totalQuantity": 59,
  "totalFreeQuantity": 4,
  "quantity": 59,
  "freeQuantity": 4
}
```

### 示例2：无赠送的订单

**订单信息**：
- 购买：1个月（2026-01-18 到 2026-02-17，共31天）
- 赠送：0天

**数据库存储**（1条记录）：
```json
{
  "validDate": "2026-01-18",
  "endDate": "2026-02-17",
  "totalQuantity": 31,
  "freeQuantity": 0,
  "remainingQuantity": 31
}
```

**接口返回**：
```json
{
  "validDate": "2026-01-18",
  "endDate": "2026-02-17",
  "totalQuantity": 31,
  "totalFreeQuantity": 0,
  "quantity": 31,
  "freeQuantity": 0
}
```

## 请求参数

### 请求体 (JSON)

```json
{
  "pageRequestModel": {
    "needTotal": true,
    "pageSize": 10,
    "pageIndex": 1,
    "skipCount": 0
  },
  "queryModel": {}
}
```

### 参数说明

#### pageRequestModel (分页参数)
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| needTotal | Boolean | 是 | 是否需要总数 |

| pageIndex | Integer | 是 | 页码（从1开始） |
| skipCount | Integer | 否 | 跳过记录数 |

| 字段名 | 类型 | 必填 | 说明 |

## 注意事项

1. 所有筛选条件均为可选，可以组合使用
2. 时间范围查询支持单边查询（只传开始时间或结束时间）
3. 有筛选条件时使用ES查询，无筛选条件时使用MySQL查询
4. ES查询失败时会自动降级到MySQL查询
5. 手机号返回时已做脱敏处理（中间4位显示为****）
6. 学员数量统计时会对studentId进行去重
7. 统计金额字段（总剩余学费、总已确认学费等）会对所有记录进行求和
8. 查询结果按订单创建时间倒序排列
9. 时间字段格式为 yyyy-MM-dd，查询时会自动补充时分秒（结束时间补充为 23:59:59）
10. 有赠送的订单在数据库中存储为两条记录，查询时会自动合并
